Why can't this run inside Claude Design / an artifact?

Artifacts render in a sandboxed environment with no route to your machine's localhost and no persistent browser storage. This file runs directly in your own browser, so it can reach Ollama/llama.cpp on localhost and save presets locally.

Multi-prompt segments (concat/combine)

Build a prompt, add it as a segment, repeat for each character/subject/scene, then combine. BREAK-join is for workflows/custom nodes that split on that keyword for regional or multi-subject conditioning — plain concatenation otherwise, this doesn't do actual regional conditioning itself.

Presets

Prompt history

Every build (manual, randomized, or AI-generated) is logged automatically, newest first, capped at 50. Stored in this browser only.

Send to ComfyUI

Requires ComfyUI launched with --enable-cors-header "*" to accept requests from a browser. Workflow, URL, and node choice are saved locally so you only paste this once. This queues generation directly — it does not wait for or show the resulting image.

Bridge to custom node (live, no export)

Pairs with the "Prompt Forge Bridge" ComfyUI node — wire its two STRING outputs into your text-encode nodes. No workflow paste needed; the node re-fetches from here on every queue.

Backup & restore

Presets, history, segments, and your ComfyUI/bridge config live only in this browser's local storage — clearing browser data, going incognito, or moving to a new machine loses all of it. Export periodically if any of this matters to you.
